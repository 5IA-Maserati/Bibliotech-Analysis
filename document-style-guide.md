# Document Style Guide

This guide ensures all documents in Bibliotech Analysis are consistent and professional.

## General Rules
- Language: English only
- File format: Markdown (.md)
- Use clear headings (#, ##, ###)
- Keep paragraphs short (3-5 lines max)
- Use bullet points for lists
- Use tables for structured data when necessary

## Naming Conventions
- Files: lowercase, hyphen-separated
  - Example: business-requirements.md, interview-transcripts.md

## Formatting Tips
- Bold important terms: **important**
- Italic for emphasis: *emphasis*
- Code snippets for commands: `command`
- Links: `[Link text](URL)`

## Version Control
- Always add date and version in the header
  - Example: `Version: 1.0 | Date: 2025-12-03`
- Use Git commit messages in present tense:
  - `Add survey template` ✅
  - `Added survey template` ❌

## Checklist Before Commit
- [ ] File name correct
- [ ] Markdown formatting correct
- [ ] Content complete and accurate
- [ ] Links work correctly
