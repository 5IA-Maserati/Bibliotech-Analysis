# Bibliotech Analysis

This repository contains all analysis work for the Bibliotech project.

## Structure

- `research/` - surveys, interviews, market research
- `requirements/` - business, functional, and non-functional requirements
- `feasibility/` - feasibility studies (technical, economic, operational)
- `templates/` - ready-to-use templates for surveys, requirements, interviews, feasibility reports
- `README.md` - this file
- `LICENSE` - MIT License
- `CONTRIBUTING.md` - contribution rules
- `document-style-guide.md` - rules for consistent document formatting

## How to Contribute

1. Always use templates from `/templates`.
2. Save new documents in the correct folder.
3. Follow the Document Style Guide.
4. Create a Pull Request for review before merging to main branch.

## Best Practices

- Write in English only.
- Use Markdown for all documents.
- Do not commit sensitive personal data.
- Follow file naming conventions: lowercase, hyphen-separated.
